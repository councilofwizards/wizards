# Conclave Realignment: Restoring Pipeline Quality on Opus 4.7

> A comprehensive change set for the conclave plugin, derived from four parallel audits (UX, pipeline orchestration,
> persona/comms machinery, drift/consistency) and one adversarial review. Every claim was checked against file contents;
> the skeptic falsified one claim, contested four fixes, and surfaced six missing dimensions â€” all are reflected below.

---

## Executive Summary

**Diagnosis.** The conclave plugin produces "dramatically less successful" runs on Opus 4.7 for three intertwined
reasons:

1. **Outright bugs that 4.6 silently worked around.** Three skills `Read` persona files that don't exist
   (`crucible-lead.md`, `castellan.md`, `atelier-lead.md`). The most-used pipeline's skeptic is pinned to a stale
   literal model id (`claude-opus-4-6`). The spec template lacks the `type: "technical-spec"` field that `build-product`
   requires. Status fields are never advanced from `draft` to `approved`, so artifact-detection skip-logic always
   re-runs completed stages.
2. **Communication scaffolding tuned for 4.6's tolerance.** The shared communication-protocol demands narrative arc
   framing ("Opening / Rising action / Climax / Resolution"), "character continuity," and "every skill invocation is a
   quest." Inside the same block, agent-to-agent mode demands terse, no-flavor prose. 4.6 indulged both registers; 4.7
   collapses toward terse and produces flat, mechanical output that reads like persona-as-decoration.
3. **Structural drift across docs and code.** `wizard-guide` advertises 16 of 26 skills. CLAUDE.md says "23 skills" on
   line 5 and "26 skills" on line 200. Validators were removed three weeks ago; the new-skill creation flow still calls
   them. Three different override conventions coexist; only one is documented; none is canonical. 4.7 spends
   context-budget reconciling drift rather than doing work.

**Treatment.** Six tiers, ordered by user-impact-per-effort:

- **Tier 0** â€” Ship-stoppers and 4.7 regression vectors (do these first; combined: ~3 hours).
- **Tier 1** â€” Artifact contract repair so pipelines stop misrouting (~4 hours).
- **Tier 2** â€” Persona/comms cleanup to align with 4.7 defaults (~6 hours).
- **Tier 3** â€” UX & discoverability so users can find the right skill (~4 hours).
- **Tier 4** â€” Pipeline structural simplification (~3 hours).
- **Tier 5** â€” Drift hygiene and lint to prevent recurrence (~2 hours).

**Tested hypothesis.** The user's hypothesis ("4.6 vs 4.7 behavioral drift exposed cracks") is supported. But the audits
also found genuine bugs that would degrade any model â€” pinning blame solely on 4.7 understates the problem.

**Single highest-leverage change.** Delete `plugins/conclave/shared/communication-protocol.md` lines 21â€“37 (the
"Narrative engagement / dramatic structure / character continuity / tone calibration" block) and re-run sync. This is
one change, cited by every audit, costs nothing, and removes the largest source of 4.7 friction.

**What this proposal preserves.** The Conclave persona system, fictional names, lore preamble, and the wizard-themed
framing are _kept_. They are scaled back from "every message is a quest beat" to "agent signs once per stage with name +
title." The user's lore investment is intentional brand identity, not noise.

**What this proposal does _not_ do.** It does not redesign the agent-team architecture, does not replace any skill, and
does not add new skills. Every change either deletes something stale, repairs a broken contract, or reshapes existing
prose to fit 4.7's grain.
